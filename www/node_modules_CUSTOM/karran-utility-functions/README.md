**** NOTE ****

    1) I advise to ensure your function is working well before moving into this utility file as it's a pain to debug and troubleshoot due to needing to continuously recompile.  
    2) Test the function on the gateway (api endpoint) AND vue3 frontend as it should work for both.


To Compile, go to this directory then run:

    npm run build

Files compiled are all things under src/ and the source file is src/index.js

Compile explained:

    rmdir dist /s /q && tsc -p tsconfig.json && tsc -p tsconfig-cjs.json && fixup.bat

        i) rmdir dist /s /q
            This just deletes the entire dist folder *NOTE* compile command breaks if no dist folder exists... just make one and re-run.
        ii) tsc -p tsconfig.json 
            Comiples for typescript (Vue3)
        iii) tsc -p tsconfig-cjs.json
            Compiles for commonjs (backend gateway)
        iv) fixup.bat
            Copies and sets package.json in each compiled source for compiler / interpreters 

Summary:
    This is specifically for functions that are to be vastly re-usable so we do not have the same commands all over our projects.

    For example... formatting dates, we could set our 5 different date formats within this file then call in all of our projects and 
    then it is standardized.  Things that are frontend only and or backend only do not belong here. Both vue3 and nodejs(gateway) should
    be able to run each of the commands.

Cache:

    Default puf.cache is initialized as a class with a 30 minute expiry.
    Able to generate a whole new cache with custom options via http://www.pseudobry.com/CacheFactory/3.0.0/Cache.html
        
        To generate a custom cache I have setup two ways...
            1)  puf.cache.new_cache(name,options)
            2)  puf.CacheService(name,options)

Date Formats:
    Callable via puf.date.X()

    mssql
        input: 1999-11-05T19:17:03:004TZ 
        output: 1999-11-05 19:17:03:004

    offset_lll
        input: date string (uses moment)
        output:  Mar 16, 2023 1:59 PM

    get_datetime(milli=false)
        output: 2023-03-16 14:02:13
        output(true): 2023-03-16 14:02:13:123
